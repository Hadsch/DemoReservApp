Ext.ux.daterange.picker.DateRange

With the Ext.ux.daterange.picker.DateRange you can select a start and an end date.
After selecting a date range, you can also use the arrow keys (left, right) to select the next or previous date range.

Specials - MoveToNext/Previous DateRange

Even take a closer look at what happens if you for example select a whole month. Now press the left/right arrow key again.
What happens, is, that you move exactly one month forward or backward. DateRange recognizes human date ranges (like weeks, months, quarters, years) and moves therefore to the next/previous human date range.

Note that the js for the example code is not minified so it is readable. See daterange-examples.js.

Ext.ux.daterange.field.DateRange

Provides a date range input field with a Ext.ux.daterange.picker.DateRange dropdown.

Specials

Type something from the following list into the field and see what happens:

  • 15 (The fifteenth of the current month is selected)
  • 5-20 (Fifth to twentieth of the current month)
  • 25.10.-25.11. (Twenty-fifth of october to twenty-fifth of november of the current year)
  • 2014 (The whole year 2014 is selected)
  • Of course this "special" inputs can be individualized

Note that the js for the example code is not minified so it is readable. See daterange-examples.js.

Ext.ux.daterange.field.DateRangeSelection

This is an extension for the Ext.ux.daterange.picker.DateRange dropdown.

Specials

What is week?
Is it from sunday to saturday, or from monday to sunday? What is if the user want to select a workweek from monday to friday? To specify what is a week in your application, you can set weekDayStart and weekDayEnd! This means if a week is selected and you call selectNextDateRange() the next week will be selected.

e.g.: weekDayStart: 1 (monday) / weekDayEnd: 5 (friday) value: Mo, 5. May - Fr, 9. May; -> expected result: Mo, 12. May - Fr, 16. May value: Mo, 5. May - Fr, 16. May; -> expected result: Mo, 19. May - Fr, 30. May
